Encapsulación

1 / 14

¿Qué es la Encapsulación?

La idea de agrupar datos (atributos) y métodos que operan sobre esos datos en una sola unidad (una clase), y restringir el acceso directo a algunos de sus componentes.

2 / 14

El Objetivo Principal

Proteger la integridad de los datos del objeto. Se busca evitar que los datos se modifiquen de forma inesperada o inválida.

3 / 14

Atributos Públicos

Por defecto, todos los atributos y métodos en Python son públicos. Se puede acceder a ellos y modificarlos desde fuera de la clase.

class Cuenta:
    def __init__(self, saldo):
        self.saldo = saldo

mi_cuenta = Cuenta(1000)
mi_cuenta.saldo = -500 # ¡Esto es posible, pero es incorrecto!
4 / 14

"Privado" por Convención: _

Un guion bajo al principio de un nombre (`_saldo`) es una convención. Le dice a otros programadores: "Esto es para uso interno, por favor no lo toques".

5 / 14

Ejemplo con _

Python no impide el acceso, pero es una señal de advertencia para el programador.

class Cuenta:
    def __init__(self, saldo):
        self._saldo = saldo # Atributo "privado"

mi_cuenta = Cuenta(1000)
# Todavía es posible, pero se considera una mala práctica
mi_cuenta._saldo = -500
6 / 14

"Fuertemente Privado": __

Dos guiones bajos (`__saldo`) activan un mecanismo llamado "Name Mangling". Python cambia el nombre internamente para evitar el acceso accidental.

7 / 14

Ejemplo con __

Intentar acceder directamente al atributo ahora da un error.

class Cuenta:
    def __init__(self, saldo):
        self.__saldo = saldo

mi_cuenta = Cuenta(1000)
# Esto dará un AttributeError:
# mi_cuenta.__saldo = -500
8 / 14

Getters y Setters

La forma de controlar el acceso es a través de métodos. "Getters" para obtener un valor y "Setters" para modificarlo (con validación).

class Cuenta:
    def __init__(self, saldo):
        self._saldo = saldo
    
    def get_saldo(self):
        return self._saldo
        
    def set_saldo(self, cantidad):
        if cantidad >= 0:
            self._saldo = cantidad
9 / 14

El Estilo "Pythónico": @property

Los decoradores `@property` y `@nombre.setter` nos permiten usar getters y setters, pero con la sintaxis simple de acceso a un atributo.

10 / 14

@property (Getter)

Convierte un método en un atributo de solo lectura.

class Persona:
    def __init__(self, nombre, apellido):
        self.nombre = nombre
        self.apellido = apellido
    
    @property
    def nombre_completo(self):
        return f"{self.nombre} {self.apellido}"

p = Persona("Juan", "Perez")
print(p.nombre_completo) # Se accede como un atributo
11 / 14

@*.setter (Setter)

Crea un método que se llama cuando se intenta asignar un valor al atributo.

class Cuenta:
    def __init__(self, saldo_inicial):
        self._saldo = saldo_inicial
    
    @property
    def saldo(self):
        return self._saldo
        
    @saldo.setter
    def saldo(self, cantidad):
        if cantidad < 0:
            raise ValueError("El saldo no puede ser negativo.")
        self._saldo = cantidad
12 / 14

Uso de Propiedades

La sintaxis es limpia y natural, pero la lógica de validación se ejecuta por detrás.

mi_cuenta = Cuenta(1000)

print(mi_cuenta.saldo) # Llama al getter (@property)

mi_cuenta.saldo = 500  # Llama al setter (@saldo.setter)

# Esto ahora lanzará un ValueError:
# mi_cuenta.saldo = -50
13 / 14

Conclusión

La encapsulación en Python se basa en convenciones (`_`) y se implementa de forma elegante con propiedades (`@property`) para proteger la integridad de los datos de un objeto.
